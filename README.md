# Employee-Performance-Mapping-SQL
SQL Project ## Project Overview SQL-based employee performance analysis system built for ScienceQtech, a data science startup. Covers the full analytics cycle from database setup to advanced query optimisation, stored functions, and views. **Tools:** MySQL | MySQL Workbench **Techniques:** DDL, DML, Joins, Subqueries, Window Functions, Stored Functions, Views, Indexing, Constraints ## Database Schema Three tables: emp_record_table (19 employees), data_science_team (data science staff). ## Key Highlights - Implemented indexing on FIRST_NAME — reduced query cost by 84% (from 2.15 full table scan to 0.35 key lookup) - Built stored function to auto-classify employees into 6 job profile tiers based on experience - Demonstrated both Window Functions and GROUP BY approaches for min/max salary analysis - Added 3 extra tasks beyond course requirement (string functions, numeric functions, table constraints) | # | Task | Technique | |---|------|-----------| | 3 | Employee list | SELECT, filtering | | 4 | Rating filter | WHERE with conditions | | 5 | Name concat | CONCAT(), alias | | 9 | Salary by role | MIN/MAX, GROUP BY + Window | | 10 | Experience rank | RANK() OVER() | | 11 | High salary view | CREATE VIEW | | 12 | Nested query | Subquery | | 13 | Job profile check | CREATE FUNCTION, CASE | | 14 | Index optimisation | CREATE INDEX, EXPLAIN | | 15 | Bonus calculation | Arithmetic formula | ## How to Run 1. Open MySQL Workbench 2. Run employee_performance_mapping.sql 3. All tables, views, functions and query outputs will be generated
